# tsParticles 4.3.0 — Master Coordination Plan

## Overview

This plan coordinates **6 features** (A, B, C, D, E, F) targeting the `4.3.0` release,
plus **H: Fluid Interaction** (under review, not in 4.3.0).
Each feature has its own detailed handover plan. This document manages sequencing, dependencies,
conflict areas, and release readiness.

---

## Hard Rule: Master Plan First, Feature Plans After

All validation steps defined in **this** document (see "Step 0: Baseline Validation" below) **must be executed and passing before any feature-specific plan** (i.e. before `BACKGROUND_CANVAS_PLAN.md`, `FLUID_INTERACTION_PLAN.md`, `GIF_SHAPE_PLAN.md`) is worked on. Even if new feature plans are added to the 4.3.0 scope mid-cycle, run this master plan's checks first, then proceed to the feature plan's steps.

This rule exists because the 4.2.0 release was burned by a `workspace:*` → hardcoded version regression discovered only at CI publish time. Running the master gate first establishes a baseline so feature work doesn't mask or introduce regressions.

## Feature F — HDR Precision (from `HDR_PRECISION_PLAN.md`)

**Target:** `@tsparticles/engine`
**Load function:** N/A

**REGOLA FERREA: ZERO NUOVE OPTIONS.** Solo calcoli interni.

**Design:** When HDR is active, skip integer rounding in `hslToRgb()` via new `hslToRgbFloat()`. Add higher-precision cache keys and float RGB random colors. **Cruciale:** rallentare le animazioni colore in HDR (`velocityFactor /= hdrVelocityDivisor`) perché lo spazio float16 ha più valori intermedi visibili — transizioni più lunghe = più dettaglio.

### Implementation phases

| Phase | Description                                                                                                                                                                                                                                             | Verification                               |
|-------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------|
| 0     | Nessuna. Costanti opzionali per precisione cache (`hdrRgbFixedPrecision`, `hdrHslFixedPrecision`).                                                                                                                                                      | Compile passes                             |
| 1     | Creare `hslToRgbFloat()` (copia di `hslToRgb` senza `Math.round()`); esportare                                                                                                                                                                          | Tests pass                                 |
| 2     | Update `getStyleFromHsl` → usa `hslToRgbFloat` in HDR; cache key più precisione in HDR; `getRandomRgbColor(hdr?)` con float per HDR                                                                                                                     | HDR banding ridotto                        |
| 3     | **Animazioni:** `updateColorValue(hdr?)` scala velocity per `sdrReferenceWhiteNits/maxNits` (inverso headroom display-p3, formula già esistente). Propagare `container.hdr` a `updateColor()` da `Particle.ts`, `PaintUpdater.ts`, `GradientUpdater.ts` | Stessa velocità visiva percepita HDR ↔ SDR |

### Files modificati

| File                                       | Change                                                                                                                                                                                            |
|--------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `engine/src/Utils/ColorUtils.ts`           | `hslToRgbFloat()`, `getStyleFromHsl` → usa `hslToRgbFloat` in HDR, `getRandomRgbColor(hdr?)`, `updateColorValue(hdr?)`/`updateColor(hdr?)` con velocity scale, cache precision 4 decimali per HDR |
| `updaters/paint/src/PaintUpdater.ts`       | Passa `container.hdr` a `updateColor()`                                                                                                                                                           |
| `updaters/gradient/src/Utils.ts`           | `updateGradient(hdr?)`, passa a `updateColor()`                                                                                                                                                   |
| `updaters/gradient/src/GradientUpdater.ts` | Passa `container.hdr` a `updateGradient()`                                                                                                                                                        |

**Skip:**
- `Constants.ts` — costanti HDR definite in ColorUtils.ts (nessuna modifica necessaria)
- `export-types.ts` — `hslToRgbFloat` già esportata via `exports.ts` → `ColorUtils.ts`
- `Particle.ts` — non chiama `updateColor()` direttamente (lo fanno gli updater)
